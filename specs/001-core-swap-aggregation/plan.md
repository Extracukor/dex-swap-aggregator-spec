# Implementation Plan: Core Swap Aggregation — Quote & Execute

**Branch**: `001-core-swap-aggregation` | **Date**: 2026-04-02
**Spec**: [spec.md](spec.md)

---

## Summary

Build the core DEX aggregation product: an off-chain routing engine that queries
Uniswap v3 and Aerodrome on Base simultaneously, selects the best route, and an
on-chain `AggregatorRouter` smart contract that executes atomic swaps with on-chain
slippage protection and protocol fee collection. A minimal React frontend allows
users to quote and execute swaps.

## Phase 2: Implementation Tasks

*Generated by `/speckit.tasks`. See [tasks.md](tasks.md).*

### Implementation Order (high-level)

1. **Smart contracts** — `AggregatorRouter` + adapters + fork tests
   (must be working before routing engine integration)
2. **Routing engine** — Fastify server + quoters + aggregator logic
   (contract address required for integration; use stub for unit tests)
3. **Frontend** — wagmi config + SwapWidget + quote hook + swap hook
   (requires routing engine endpoint for full integration)

### Key Constraints for Task Execution

- Fork tests MUST pass on a pinned Base mainnet block before any deployment.
- `contracts/routing-api.md` is the source of truth for all request/response shapes —
  routing engine and frontend MUST NOT deviate from it.
- The `swap()` function ABI is frozen; any change requires a constitution amendment.
- `minAmountOut` passed to `swap()` = `bestAmountOut × (1 - slippageBps/10000)`,
  calculated off-chain by the frontend using the quote.
- ETH swaps: frontend sends `msg.value = amountIn`; contract wraps to WETH internally.
